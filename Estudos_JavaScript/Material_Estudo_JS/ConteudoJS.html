Oque é

JavaScript é uma linguagem de programação de alto nível, interpretada e dinâmica. Foi originalmente desenvolvida para adicionar interatividade às páginas da web, mas hoje em dia é amplamente utilizada em diversas áreas de desenvolvimento de software, incluindo aplicativos web, aplicativos móveis, jogos, servidores e até mesmo em hardware.

Ela é executada no lado do cliente, o que significa que o código JavaScript é executado no navegador da web do usuário, não no servidor.

O JavaScript foi criado pela Netscape Communications Corporation, uma empresa de software norte-americana que desempenhou um papel fundamental no desenvolvimento da web nas décadas de 1990 e 2000. Brendan Eich, um engenheiro de software da Netscape, foi o responsável pelo desenvolvimento do JavaScript em 1995.

Alguns exemplos de programas ou softwares que utiliza JavaScript incluem:

  • Navegadores da Web: Google Chrome, Mozilla Firefox, Microsoft Edge, Safari, entre outros, todos usam JavaScript para criar páginas da web interativas.
  • Frameworks e bibliotecas JavaScript: React.js, Angular.js, Vue.js, Node.js, jQuery, Express.js, entre outros, são usados para desenvolver aplicativos web e servidores.
  • Aplicativos de desktop: Ferramentas como Electron permitem que os desenvolvedores criem aplicativos de desktop usando tecnologias web, incluindo JavaScript.
  • Aplicativos móveis: Frameworks como React Native permitem o desenvolvimento de aplicativos móveis usando JavaScript.
  • Jogos: Muitos jogos web e móveis são desenvolvidos usando HTML5, CSS e JavaScript.

ECMAScript

É o padrão internacional para JavaScript. Ele define a especificação da linguagem, incluindo a sintaxe, tipos de dados, instruções, controle de fluxo, e outros elementos que são essenciais para escrever código JavaScript. Em resumo, ECMAScript é a especificação técnica que descreve como a linguagem JavaScript deve funcionar.

JavaScript é a implementação concreta da linguagem com base na especificação do ECMAScript. Sempre que falamos em JavaScript, estamos nos referindo a uma implementação específica dessa linguagem, que segue as diretrizes e especificações definidas pelo padrão ECMAScript.

Client X Server

Lado do Cliente: Quando um usuário acessa uma página da web, o navegador solicita ao servidor web o conteúdo da página, incluindo HTML, CSS e JavaScript. O navegador interpreta o HTML para criar a estrutura da página, aplica o CSS para estilizá-la e executa o JavaScript para adicionar interatividade. O JavaScript pode validar formulários, manipular elementos da página e realizar solicitações assíncronas ao servidor, tudo sendo executado no computador do usuário, sem necessidade de interação adicional com o servidor.

Lado do Servidor: O lado do servidor é onde o código é executado antes de ser enviado para o navegador do usuário. Linguagens como PHP, Python, Ruby, Java e Node.js são comumente usadas para processar solicitações, acessar bancos de dados e gerar conteúdo dinâmico. O lado do servidor pode renderizar páginas da web com base em dados específicos do usuário e também executar JavaScript, permitindo aos desenvolvedores criar aplicativos web completos em uma única linguagem de programação.

Varíaveis

Variáveis asrmazenam um tipo de valor na memória:

  • String é Sequência de carcateres
  • Number representa os números inteiros
  • Floot representa os números decimais, utiliza-se o ponto e não a vírgula
  • Booleano é uma estrutura de decisão, retorna um valor de dois(true ou false)
  • Constante é um valor inalterável
  • Let declara uma variavel levando em conta o escopo onde foi definida

Laços de repetição

São conhecidos como laços de iteração ou simplesmente loops, são comandos que permitem a iteração de código, fazendo que o código seja repetido(exemplos abaixo).

  • Switch: Estrutura de controle de fluxo que avalia uma expressão, dependendo do valor, executa um bloco de código correspondente.
  • If/If else: Usada para tomar decisões com base em condições, normalmente utiliza os operadores para as comparações, vericando por exemplo se uma varíavel é maior ou igual a outra varíavel.
  • For: Uitlizadop para executar um bloco de código várias vezes, possui três partes: inicialização, condição de continuação e expressão de iteração.
  • While: Usado para repetir um bloco de código enquanto uma condição for verdadeira.

Array

Estrutura de dados que armazena uma coleção ordenada de valores. Os arrays podem conter diversos tipos de dados, como números, strings, objetos e até mesmo outros arrays, cuja posição são os indíces e os valores são os dados, inícia no zero (Array = [345]-Posição 0 = 3, Posição 1 = 4, Posição 2 = 5);

Funções:

São trechos de código que executam um comando e retornam um valor, pode ser usado para fazer calculos, validações e conversões, podendo ser reutilizados.
as funções podem receber paramêtros(respeitar paramêtros ao chamar função) e para acessalos podemos usar argumentos ou args, os paramêtros são as informações que a função precisa

  • Functions declaration (Função de declaração): Declaração de função usando a palavra-chave function, pode ser chamada antes de ser declarada (hoisting), possui um nome que é usado para chamá-la.
  • Functions expression (Função de expressão): A função é atribuia a uma varíavel disponível apenas depois da atribuição e pode ter ou não um nome.
  • Arrow Functions (Função de flecha): Sintaxe mais curta usadando a seta (=>), o contexto (this) é herdado do escopo pai, útil para funções curtas ou expressivas.
  • Functions constructor (Função construtora): Cria objetos usando a palavra-chave NEW, usa o this para referenciar o objeto atual, menos eficiente em comparação com outras abordagens.
  • Generator Functions (Função geradora): Introduzido para simplificar a criação de iteradores, usa a palavra-chave Function e pode pausar a execução e ser retornado.

Eventos

São ações ou ocorrências que acontecem durante a execução de um programa. No contexto da web e do JavaScript, eventos referem-se principalmente às interações do usuário com uma página web, abaixo alguns exemplos:

  • Click Mouse: Acionado ao clicar em um botão.
  • Precionar teclaa: acciona ao precionar uma tecla do teclado.
  • Carregamento da página: Quando a página é totalmente carregada.
  • Envio de formulário: Quando um formulário é enviado.
  • Movimento do Mouse: Quando o mouse se move sobre um elemento.

Objeto

O objeto é uma entidade independente, com propriedades que definem suas características e tipos como number e bool, o objeto sempre representa a classe, instância é o nome desse processo

Classe

É um conjunto de atríbutos("similar a var") e métodos("similar a funções"), ela engloba todos os assuntos em comum, método são ações que serão executadas. A varíavel na classe é o atríbuto porque recebem alguns recursos a mais, podendo ser chamado dentro da classe onde quiser, ou quem chama eles, o método é como as funções com a diferença que não é criado com a palavra fuunction, ele precisa estar dentro da classe para executar algo, podendo ser chamado de outros lugares da classe, para isso utilizamos o comando THIS(comando interno usado para referênciar a um atríbuto ou método).

Para usar a classe utiliza-se a palavra reservada NEW, cria uma nova intância da classe(Intância é quando o objeto representa uma classe), existe classes nativas com a classe date que é utilizada para trabalhar com datas e horas.

MVC

O conceito de MVC é de uma organzição entre os meus dados tratados(Model),a interface do usuário(View) e as regras de negócio que podem ou não acontecer(Controller)

Construtor

São funções especiais utilizadas para criar e inicializar objetos. Eles são chamados quando você cria uma nova instância de uma classe ou de um tipo de objeto específico. A palavra-chave new é usada para chamar um construtor e criar um novo objeto.

Encapsulamento

Funciona com a utilização de modificadores de acesso para restringir o acesso aos atributos e aos métodos de um objeto. Garante que somente as classes apropriadas possam acessar as informações e ajuda a assegurar que os atributos e os métodos sejam usados de forma consistente e previsível, atributos podem ter encapsulamento(Acesso ao atributos), e ao criar o atributo é importante criar o get e set para ser controlado, tipos comuns de encapsulamento

  • Public: Todos acessam;
  • Protected: Acessam apenas atributos e métodos da mesma classe e classe pai;
  • Private: Acessam somente atributos e métodos da própria classe;

Métodos Getters e Setters

Permitem definir como acessar os valores,Esses métodos são úteis para criar interfaces mais controladas e seguras ao trabalhar com objetos.
O método GETTERS(get) é responsavel por obter o valor de uma propriedade, é comum prefixar propriedades privadas com um sublinhado _ para indicar que elas não devem ser acessadas diretamente usados para acessar valores de propriedades de maneira controlada. Eles podem realizar operações antes de retornar o valor.
O método SETTERS(set) é resposável por definir o valor de uma propriedade, é usado para realizar ações ou validações antes de atribuir um novo valor a uma propriedade.

DOM

Document object model, estrutura como o HTML é formado, ao interagir com o conteúdo para o usuário utilizando o DOM para manipulá-lo. O DOM manipula o documento ao contrário do BOM(Browser Object Model) que manipula o browser, o BOM é a janela do navegador, exemplo: Barra de endereços, minimizar tela, entre outros.
Cada tag criada em html vira objeto da coleção HTMLDocument.

Oque um desenvolvedor JS precisa saber?

  1. Sintaxe Básica e Estrutura de Controle: Declaração de variáveis, operadores, estruturas de controle de fluxo (como if, else, switch, loops), manipulação de arrays e objetos.
  2. Funções e Escopo: Entendimento de como as funções funcionam em JavaScript, incluindo declaração, expressões de função, escopo de variáveis, argumentos de função e retorno de valores.
  3. Programação Orientada a Objetos (POO): Conhecimento sobre como trabalhar com objetos em JavaScript, incluindo criação de objetos, propriedades, métodos, encapsulamento, herança e polimorfismo.
  4. Manipulação do DOM: Habilidade para interagir com o Document Object Model (DOM) para manipular elementos HTML, criar e modificar elementos, alterar estilos e atributos, manipular eventos do usuário, etc.
  5. Assincronismo e Promises: Compreensão de como trabalhar com operações assíncronas em JavaScript, incluindo callbacks, Promises, async/await, e como lidar com a execução de código assíncrono de forma eficiente.
  6. Trabalho com APIs: Capacidade de consumir e trabalhar com APIs externas, seja através de XMLHttpRequest, Fetch API, ou bibliotecas/frameworks como Axios.
  7. Manipulação de Strings e Arrays: Habilidade para trabalhar eficientemente com strings e arrays em JavaScript, incluindo métodos de manipulação de strings, métodos de array (map, filter, reduce, etc.), e outras operações comuns.
  8. Gerenciamento de Erros e Depuração: Conhecimento sobre como lidar com erros em JavaScript, incluindo uso de try/catch, throw, e ferramentas de depuração do navegador para identificar e corrigir problemas no código.
  9. Gerenciamento de Estado: Familiaridade com diferentes abordagens para gerenciamento de estado em aplicações JavaScript, incluindo estado local, cookies, armazenamento local (localStorage/sessionStorage), e frameworks/bibliotecas de gerenciamento de estado como Redux, Vuex, MobX, etc.
  10. Bons Princípios de Codificação: Conhecimento sobre boas práticas de codificação em JavaScript, incluindo legibilidade do código, modularização, reutilização de código, organização de arquivos, etc.
  11. Frameworks e Bibliotecas: Experiência com pelo menos um dos principais frameworks ou bibliotecas JavaScript, como React.js, Angular, Vue.js, Node.js, Express.js, etc., dependendo das áreas de interesse e especialização.
  12. Testes Unitários e Integração: Compreensão de como escrever testes unitários e de integração para garantir a qualidade do código JavaScript, usando ferramentas como Jest, Mocha, Jasmine, etc.
  13. Performance e Otimização: Conhecimento sobre técnicas de otimização de desempenho em JavaScript, incluindo redução de renderizações, redução de tempo de carregamento, lazy loading, etc.
  14. Segurança: Conscientização sobre práticas de segurança em JavaScript, incluindo prevenção de ataques comuns como injeção de código, XSS (Cross-Site Scripting), CSRF (Cross-Site Request Forgery), etc.
  15. Ferramentas de Desenvolvimento: Experiência com ferramentas de desenvolvimento JavaScript, como editores de código (VSCode, Sublime Text, etc.), sistemas de controle de versão (Git), gerenciadores de pacotes (npm, yarn), bundlers (Webpack, Parcel), entre outros.
